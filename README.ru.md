# go-pkg

go-pkg — небольшая коллекция переиспользуемых Go-пакетов и лёгких примеров, предназначенная как стартовая библиотека для сервисных проектов. Фокус на прагматичных настройках, небольшой поверхности API и понятных, тестируемых утилитах.

Основные пакеты
- config — загрузка конфигурации из YAML/ENV с поддержкой значений по умолчанию и duration.
- logger — тонкая обёртка над Go slog с удобными helper‑методами для структурированного логирования.
- database — утилиты для работы с SQL БД, мониторинга соединений и миграций.
- metrics — модульные хелперы для метрик HTTP и БД.
- migrations — загрузка SQL‑миграций из fs.FS и безопасный раннер миграций.

Быстрый старт
1. Прогнать тесты:
   cd go-pkg && go test ./...
2. Запустить пример (без БД):
   go run ./examples/basic
3. Advanced пример (требуется Postgres):
   go run ./examples/advanced
   Настройте подключение через конфиг примера или переменные окружения.

Миграции
- Единая, каноническая реализация мигратора находится в pkg/database/migrator.go.
  Поддерживает:
  - Программные миграции (интерфейс Migration).
  - Запись применённых версий в таблицу `schema_migrations`.
  - Откат последней применённой миграции.
  - Загрузку SQL‑миграций из embedded fs.FS через `LoadMigrationsFromFS`.
  - Вспомогательную функцию `RunDirectoryMigrations` для применения `*.up.sql` из директории.
- pkg/migrations предоставляет тонкие alias/обёртки, делегирующие вызовы в pkg/database для обратной совместимости.

Логирование
- pkg/logger даёт простой wrapper для структурированного логирования с helper‑ами:
  - WithComponent, WithError, WithRequestID, WithTraceID, WithContext.
  - Удобные функции: Request, Business, HTTPRequest, DatabaseOperation, Performance.
- Формат по умолчанию: JSON. Уровень по умолчанию: info.
- Используйте `logger.InitGlobal` для инициализации глобального логгера либо создавайте локальные экземпляры через `logger.New`.

Тестирование
- Unit‑тесты используют sqlmock для тестирования кода, работающего с БД.
- Запуск всех тестов с покрытием:
  cd go-pkg && go test ./... -cover

Контрибуция
- Предпочтительны небольшие целевые изменения.
- Держите публичный API минимальным и документированным.
- Добавляйте unit‑тесты для новой логики и прогоняйте `go vet` / `golangci-lint`, если доступно.

Заметки и рекомендации
- Проект ориентирован на компактность и готовность к публикации как утилитарная библиотека. Для жёстких гарантий миграций (ACID/транзакции/стратегии отката) используйте программный мигратор в pkg/database, а не простые раннеры по файловой директории.
- Рекомендуется добавить CI (GitHub Actions) для автоматического прогона тестов и проверки покрытия.
